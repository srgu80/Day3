# ūüü¶ **1. What Is Semi‚ÄĎStructured Data?**

Semi‚ÄĎstructured data is **not fully tabular**, but it still has **some organizational structure** such as:

- key‚Äďvalue pairs  
- nested objects  
- arrays  
- flexible schema  

### Examples:
- JSON  
- XML  
- YAML  
- Avro  
- Parquet (schema‚ÄĎon‚ÄĎread)  

### Why it matters:
Modern applications (APIs, microservices, logs, events, IoT) generate **JSON‚ÄĎbased data**, and relational systems must store and query it efficiently.

---

# ūüü¶ **2. Why Store JSON in Relational Databases?**

Relational systems (MySQL, PostgreSQL, SQL Server, Oracle) now support JSON because:

- Microservices exchange JSON payloads  
- Event‚ÄĎdriven systems store logs in JSON  
- Cloud apps (AWS/Azure) use JSON for config/state  
- Data lakes ingest JSON before ETL  
- Flexible schema reduces friction during ingestion  

**Hybrid workloads** need both:
- relational consistency  
- semi‚ÄĎstructured flexibility  

---

# ūüü¶ **3. JSON Storage Options in RDBMS**

### **A. Store JSON as TEXT/VARCHAR**
- Simple  
- No validation  
- No indexing  
- Slow for querying  

### **B. Store JSON in Native JSON Column Type**
Supported in:
- MySQL  
- PostgreSQL  
- SQL Server (JSON functions only)  
- Oracle (JSON data type in 21c+)  

**Benefits:**
- Validation  
- Indexing  
- Efficient storage  
- JSON functions for querying

# ūüü¶ **4. Creating a JSON Table (MySQL Example)**

```sql
CREATE TABLE orders (
    order_id INT PRIMARY KEY,
    customer_id INT,
    order_data JSON
);
```

Insert JSON:

```sql
INSERT INTO orders VALUES
(1, 101, '{"items":[{"product":"Laptop","qty":1}], "total":1200, "status":"shipped"}'),
(2, 102, '{"items":[{"product":"Mobile","qty":2}], "total":1500, "status":"processing"}');
```

---

# ūüü¶ **5. Querying JSON Data**

### **Extract a JSON field**
```sql
SELECT 
    order_id,
    order_data->'$.status' AS status
FROM orders;
```

### **Extract nested values**
```sql
SELECT
    order_id,
    order_data->'$.items[0].product' AS first_product
FROM orders;
```

### **Using JSON_EXTRACT**
```sql
SELECT
    JSON_EXTRACT(order_data, '$.total') AS total_amount
FROM orders;
```

---

# ūüü¶ **6. Updating JSON Fields**

```sql
UPDATE orders
SET order_data = JSON_SET(order_data, '$.status', 'delivered')
WHERE order_id = 2;
```

---

# ūüü¶ **7. Searching Inside JSON**

### Find orders where product = 'Laptop'

```sql
SELECT *
FROM orders
WHERE JSON_SEARCH(order_data, 'one', 'Laptop') IS NOT NULL;
```

---

# ūüü¶ **8. JSON Indexing (Performance Optimization)**

### **Generated Column + Index**
```sql
ALTER TABLE orders
ADD total_amount INT 
    GENERATED ALWAYS AS (JSON_EXTRACT(order_data, '$.total')) STORED;

CREATE INDEX idx_total_amount ON orders(total_amount);
```

This makes JSON queries **fast and index‚ÄĎfriendly**.

---

# ūüü¶ **9. JSON Validation**

### Check if JSON is valid
```sql
SELECT JSON_VALID(order_data) FROM orders;
```

### Reject invalid JSON on insert
MySQL JSON column automatically enforces this.

# ūüü¶ **1. Create Source Table (Intentionally Nested JSON + Arrays)**  
We‚Äôll build a realistic e‚ÄĎcommerce dataset.

```sql
CREATE TABLE orders_json (
    order_id INT,
    customer_id INT,
    order_payload JSON
);

INSERT INTO orders_json VALUES
(1, 101,
 '{
    "order_date": "2024-01-01",
    "shipping": {"city": "Bengaluru", "pincode": 560001},
    "items": [
        {"product": "Laptop", "qty": 1, "price": 1200},

 }'),

(2, 102,
```

# ūüü¶ **2. Extract Simple Nested Fields (JSON_EXTRACT)**

### **Goal:** Extract order_date, city, payment mode.

```sql
SELECT
    order_id,
    JSON_EXTRACT(order_payload, '$.order_date') AS order_date,
    JSON_EXTRACT(order_payload, '$.shipping.city') AS city,
    JSON_EXTRACT(order_payload, '$.payment.mode') AS payment_mode
FROM orders_json;
```

### What learners observe  
- Dot notation for nested objects  
- JSON paths (`$.field.subfield`)

# ūüü¶ **6. Flatten Arrays Using JSON_TABLE (MySQL 8+)**  
This is the **most important transformation** for real‚ÄĎworld pipelines.

### **Goal:** Convert each item in the array into its own row.

```sql
SELECT
    o.order_id,
    o.customer_id,
    jt.product,
    jt.qty,
    jt.price
FROM orders_json o,
JSON_TABLE(
    o.order_payload,
    '$.items[*]' 
    COLUMNS (
        product VARCHAR(50) PATH '$.product',
        qty INT PATH '$.qty',
        price INT PATH '$.price'
    )
) AS jt;
```

### What learners observe  
- `JSON_TABLE` expands arrays into rows  
- Each array element becomes a row  
- Perfect for analytics, reporting, ETL  

---

# ūüü¶ **7. Flatten + Join With Parent Fields**

# ūüü¶ **12. Full Flattening Pipeline (Capstone Exercise)**  
### **Goal:** Produce a fully flattened order_items table.

```sql
SELECT
    o.order_id,
    o.customer_id,
    o.order_payload->'$.order_date' AS order_date,
    o.order_payload->'$.shipping.city' AS city,
    jt.product,
    jt.qty,
    jt.price,
    (jt.qty * jt.price) AS line_total,
    o.order_payload->'$.payment.mode' AS payment_mode,
    o.order_payload->'$.payment.status' AS payment_status
FROM orders_json o,
JSON_TABLE(
    o.order_payload,
    '$.items[*]'
    COLUMNS (
        product VARCHAR(50) PATH '$.product',
        qty INT PATH '$.qty',
        price INT PATH '$.price'
    )
) jt;
```

This produces a **fully normalized, analytics‚ÄĎready dataset**.
